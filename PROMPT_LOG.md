## Задание средней сложности: 1. Определение агентов и их ролей
### Промпт 1
**Инструмент:** OpenCode \
**Промпт:** Предметная область: Управление производством. Типы агентов: Планирование загрузки, Контроль качества,
Управление запасами, Диспетчеризация. Опиши агентов: описать их: роль, входные данные, выходные данные, бизнес-правила \
**Результат:** 

Агент 1: Планирование загрузки (Load Planner) 
* Роль:	Составление оптимального расписания загрузки производственных мощностей на основе поступивших заказов
* Входные данные:	{ order_id, product, quantity, deadline, priority }
* Выходные данные:	{ order_id, schedule: [{ machine, start_time, end_time }], total_time, utilization_pct, feasible }
* Бизнес-правила:	1. Приоритетные заказы выполняются раньше обычных. 2. Загрузка станка не может превышать 100%. 3. Если deadline нереалистичен → feasible: false с рекомендацией нового срока. 4. Между операциями на одном станке — технологический перерыв 15 мин. 5. Один заказ может быть разбит на несколько станков, если это сокращает время.

Агент 2: Контроль качества (Quality Control) 
* Роль:	Проверка выпущенной продукции на соответствие стандартам качества
* Входные данные:	{ batch_id, product, quantity, measurements: [{ param, value, unit }], order_id }
* Выходные данные:	`{ batch_id, status: "passed"
* Бизнес-правила:	1. Допуск по каждому параметру — ±5% от номинала. 2. Если defect_rate > 10% → rejected, вся партия бракуется. 3. Если 3% < defect_rate ≤ 10% → rework, требуется доработка. 4. Выборочный контроль — 20% от партии, но не менее 10 единиц. 5. Для критических параметров (указаны в product spec) допуск ±1%.

Агент 3: Управление запасами (Inventory Manager)
* Роль:	Мониторинг и управление складскими запасами сырья, материалов и готовой продукции
* Входные данные:	`{ request_type: "check"
* Выходные данные:	`{ request_id, status: "ok"
* Бизнес-правила:	1. Минимальный страховой запас — 15% от среднемесячного расхода. 2. Если available < required + safety_stock → shortage. 3. При shortage автоматически формируется заказ поставщику (reorder). 4. Время поставки зависит от материала (от 1 до 14 дней). 5. Резерв под заказ снимает доступное количество на складе.

Агент 4: Диспетчеризация (Dispatcher)
* Роль:	Назначение производственных заданий на конкретные линии/участки и контроль выполнения
* Входные данные:	{ order_id, schedule: [{ machine, task, duration }], priority, start_after }
* Выходные данные:	{ order_id, dispatch_id, lines: [{ line, task, status, actual_start, expected_end }], overall_status }
* Бизнес-правила:	1. Задание назначается на линию с наименьшей текущей загрузкой. 2. Одна линия выполняет одно задание за раз. 3. Если линия занята → задание ставится в очередь линии. 4. Критические заказы (high priority) могут прерывать выполнение обычных. 5. По завер
